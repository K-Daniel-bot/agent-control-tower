# AI 원격 제어 시스템 (AI Remote Control System)

## Agent Control Tower — "원격" 탭 시스템 계획서

**문서 버전:** v1.0
**작성일:** 2026-03-06
**프로젝트:** Agent Control Tower (Next.js 15 / React 19 / TypeScript 5)
**통합 위치:** `src/components/remote/` → Header TabType `'remote'`

## 1. 시스템 개요

### 1.1 목적

AI 에이전트가 사용자의 OS를 실시간으로 원격 제어하는 시스템이다. 커서 이동, 클릭, 키보드 입력, 웹 브라우징까지 모든 OS 기능을 AI가 라이브 액션 모드로 수행한다. 보안이 핵심이며, 모든 작업은 심각도(낮음/중간/높음)에 따라 사용자 승인을 거친다. AI는 음성으로 사용자와 소통하며, 항상 "주인님"이라 호칭한다.

### 1.2 핵심 원칙

- **라이브 액션**: 실시간 커서 제어, 클릭, 타자 — 사용자가 화면에서 직접 확인
- **고속 실행**: 지연 없는 빠른 속도의 자동화
- **철저한 보안**: 샌드박스 환경, 민감 영역 접근 차단, 심각도 기반 승인
- **시스템 감시**: AI가 컴퓨터 전체를 모니터링하며 안전 유지
- **음성 인터페이스**: AI가 음성으로 상태 보고 및 승인 요청 ("주인님")

## 3. 핵심 기능 상세

### 3.1 라이브 액션 모드 (Live Action Mode)

AI가 OS를 직접 제어하는 실시간 모드이다.

**기능 범위:**

| 기능 | 설명 | 구현 방식 |
|------|------|-----------|
| 커서 이동 | 마우스 포인터를 목표 좌표로 이동 | `robotjs` / `nut.js` 네이티브 바인딩 |
| 클릭 | 좌클릭, 우클릭, 더블클릭 | Input Driver → OS API |
| 드래그 | 드래그 앤 드롭 조작 | 좌표 시퀀스 + 마우스 다운/업 |
| 키보드 입력 | 텍스트 타자, 단축키 조합 | 가상 키보드 이벤트 주입 |
| 스크롤 | 화면/요소 스크롤 | 휠 이벤트 시뮬레이션 |
| 화면 캡처 | 실시간 스크린 스트리밍 | `screenshot-desktop` + WebSocket |
| 앱 실행 | 프로그램 열기/닫기 | `child_process.exec` (샌드박스 내) |
| 파일 관리 | 파일 복사/이동/삭제/생성 | Node.js `fs` (권한 제어 하) |
| 웹 브라우징 | 보안 브라우저로 웹 접속 | Puppeteer/Playwright (샌드박스) |

**속도 최적화 전략:**

- 네이티브 OS API 직접 호출 (JavaScript 바인딩)으로 지연 최소화
- 화면 캡처: 30fps 이상 WebSocket 스트리밍, 변경 영역만 delta 전송
- 명령 배치 처리: 연속 액션을 묶어 한 번에 전송
- 액션 예측 파이프라인: AI가 다음 행동을 미리 계산하여 대기시간 제거

### 3.2 심각도 기반 승인 시스템 (Severity-Based Approval)

모든 AI 액션은 심각도 분류를 거쳐 사용자 승인 프로세스를 따른다.

```
AI 액션 요청
    │
    ▼
┌─────────────────────┐
│  심각도 분류 엔진    │
│  (Severity Classifier)│
└─────────┬───────────┘
          │
    ┌─────┼─────┐
    ▼     ▼     ▼
  낮음   중간   높음
    │     │     │
    ▼     ▼     ▼
  자동   음성   음성+시각
  실행   승인   승인+확인
         요청   이중검증
```

**심각도 분류 기준:**

#### 낮음 (Low) — 자동 실행 (로그만 기록)
- 마우스 이동 및 스크롤
- 화면 캡처 및 정보 읽기
- 시스템 상태 조회
- 앱 내 탐색 (탭 전환, 메뉴 열기)
- 클립보드 읽기 (쓰기 제외)

**AI 음성 예시:**
> "주인님, 현재 화면을 확인하고 있습니다."

#### 중간 (Medium) — 음성 승인 요청
- 텍스트 입력 및 편집
- 파일 열기/생성/복사
- 앱 실행/종료
- 웹사이트 접속
- 시스템 설정 읽기
- 클립보드 쓰기

**AI 음성 예시:**
> "주인님, 메모장에 회의록을 작성하려 합니다. 진행해도 될까요?"

**승인 방식:**
- 음성 응답: "응", "해", "진행해" → 승인
- 음성 응답: "아니", "취소", "멈춰" → 거부
- UI 버튼: 승인/거부 클릭
- 5초 내 미응답 → 대기 (반복 안내)

#### 높음 (High) — 이중 검증 (음성 + UI 확인)
- 파일 삭제/이동/덮어쓰기
- 시스템 설정 변경
- 네트워크 설정 변경
- 소프트웨어 설치/삭제
- 레지스트리/커널 파라미터 수정
- 관리자 권한 필요 작업
- 결제/금융 관련 페이지 접근
- 개인정보가 포함된 파일 접근
- 외부 서버로의 데이터 전송

**AI 음성 예시:**
> "주인님, 높은 심각도 작업입니다. 'C:\Users\Documents\중요문서.xlsx'를 삭제하려 합니다. 음성과 화면의 확인 버튼 모두로 승인해 주십시오."

**승인 방식:**
- 음성 승인 + UI 확인 버튼 클릭 (둘 다 필요)
- 상세 영향 분석 팝업 표시
- 실행 전 3초 카운트다운 (취소 가능)

#### 차단 (Blocked) — 절대 실행 불가
- OS 핵심 시스템 파일 수정 (커널, 부트로더)
- 보안 소프트웨어 비활성화
- 루트/관리자 비밀번호 변경
- 디스크 포맷/파티션 변경
- BIOS/UEFI 설정 접근

**AI 음성 예시:**
> "주인님, 해당 작업은 시스템 보안 정책에 의해 차단된 영역입니다. 수동으로 진행해 주시기 바랍니다."

### 3.3 시스템 감시 (System Monitoring)

AI가 컴퓨터 전체를 상시 감시하며 이상 징후를 탐지한다.

**감시 항목:**

| 카테고리 | 감시 대상 | 탐지 예시 |
|----------|-----------|-----------|
| 프로세스 | CPU/RAM 이상 사용, 알 수 없는 프로세스 | 크립토마이너, 멀웨어 프로세스 |
| 네트워크 | 비정상 아웃바운드 트래픽, 알 수 없는 연결 | C2 서버 통신, 데이터 유출 |
| 파일시스템 | 대량 파일 변경, 시스템 파일 변조 | 랜섬웨어, 루트킷 |
| 레지스트리 | 시작 프로그램 변경, 보안 설정 변조 | 악성 자동실행 등록 |
| 저장장치 | 디스크 건강 상태, 용량 부족 | SMART 경고, 디스크 풀 |
| 성능 | 시스템 온도, 팬 속도, 배터리 | 과열 위험, 배터리 열화 |

**이상 탐지 → 대응 흐름:**
```
이상 징후 탐지
    │
    ▼
┌────────────────┐
│ 위협 분석 엔진 │──▶ 심각도 판정
└───────┬────────┘
        │
        ├─── 정보성 ──▶ "주인님, 메모리 사용률이 85%입니다." (로그)
        │
        ├─── 경고 ────▶ "주인님, 알 수 없는 프로세스가 발견되었습니다.
        │                 종료할까요?" (승인 요청)
        │
        └─── 위험 ────▶ "주인님! 긴급 상황입니다. 랜섬웨어가 의심됩니다.
                          즉시 네트워크를 차단하겠습니다!" (긴급 대응 + 통보)
```

### 3.4 보안 샌드박스 (Security Sandbox)

모든 AI 원격 조작은 샌드박스 환경에서 실행된다.

**샌드박스 계층:**

```
┌──────────────────────────────────────┐
│         사용자 OS (Host)              │
│  ┌────────────────────────────────┐  │
│  │     보안 게이트웨이            │  │
│  │  (접근 제어 / 권한 관리)       │  │
│  │  ┌──────────────────────────┐  │  │
│  │  │   AI 실행 샌드박스       │  │  │
│  │  │  ┌────────┐ ┌────────┐  │  │  │
│  │  │  │Input   │ │Browser │  │  │  │
│  │  │  │Driver  │ │(격리)  │  │  │  │
│  │  │  └────────┘ └────────┘  │  │  │
│  │  │  ┌────────┐ ┌────────┐  │  │  │
│  │  │  │File    │ │Process │  │  │  │
│  │  │  │Access  │ │Control │  │  │  │
│  │  │  │(제한)  │ │(격리)  │  │  │  │
│  │  │  └────────┘ └────────┘  │  │  │
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

**보안 정책:**
- 파일시스템: 화이트리스트 기반 접근 — 허용된 경로만 접근 가능
- 네트워크: URL 필터링 + TLS 검증 — 악성 사이트 자동 차단
- 프로세스: AI가 실행할 수 있는 프로세스 목록 사전 정의
- 브라우저: 격리된 Chromium 인스턴스 — 호스트 브라우저 데이터 격리
- 입력: 모든 입력 이벤트 감사 로그 기록

### 3.5 음성 인터페이스 (Voice Interface)

AI는 항상 음성으로 사용자와 소통한다.

**음성 엔진:**
- TTS (Text-to-Speech): 한국어 자연어 음성 합성 — 친근하고 정중한 톤
- STT (Speech-to-Text): 사용자 음성 명령 인식
- Wake Word: "어시스턴트" 또는 커스텀 호출어

**화법 규칙:**
- 항상 "주인님"으로 호칭
- 존댓말 사용 (합쇼체)
- 작업 시작 시: "주인님, [작업내용]을 시작하겠습니다."
- 승인 요청 시: "주인님, [작업내용]을 진행해도 될까요?"
- 완료 시: "주인님, [작업내용]을 완료했습니다."
- 오류 시: "주인님, 문제가 발생했습니다. [상세내용]"
- 위험 감지 시: "주인님! [위험내용]이 감지되었습니다. 즉시 대응하겠습니다."

### 4.1 원격 탭 전체 레이아웃

```
┌─────────────────────────────────────────────────────────────────────┐
│  [Header] 관제센터 | 터미널 | 서버 | ... | ★원격★ | 설정           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
```

```typescript
interface RemoteState {
  // 연결 상태

  };

    readonly x: number;
    readonly y: number;

  };

  readonly approvalQueue: readonly ApprovalRequest[];
  readonly approvalHistory: readonly ApprovalRecord[];

  // 시스템 모니터
  readonly systemMetrics: {
    readonly cpu: number;
    readonly ram: { used: number; total: number };
    readonly disk: { used: number; total: number };
    readonly network: { upload: number; download: number };
  };
  readonly threats: readonly ThreatAlert[];
  readonly processes: readonly ProcessInfo[];

  // 음성
  readonly voiceState: {
    readonly isListening: boolean;
    readonly isSpeaking: boolean;
    readonly lastMessage: string;
    readonly waveform: readonly number[];
  };

  // 보안
  readonly securityLevel: 'normal' | 'elevated' | 'critical';
  readonly auditLog: readonly AuditEntry[];
  readonly accessControlList: readonly ACLRule[];
}

interface ApprovalRequest {
  readonly id: string;
  readonly timestamp: number;
  readonly severity: 'low' | 'medium' | 'high' | 'blocked';
  readonly action: string;
  readonly description: string;
  readonly target: string;
  readonly estimatedImpact: string;
  readonly status: 'pending' | 'approved' | 'denied' | 'expired';
  readonly voiceMessage: string; // AI가 읽을 음성 메시지
}
```
